Texniki Arxitektura

Necə İşləyir

3 mərhələli skan prosesi Instagram-ın öz API-larından real vaxt məlumatları toplayır.

1
Mərhələ 1 — Following Siyahısı
GraphQL query ilə bütün izlədikləriniz yüklənir. Hər 5 sorğudan sonra avtomatik fasilə verilir.
GET /graphql/query/?query_hash=3dec7e2c...
2
Mərhələ 2 — Followers Siyahısı
Sizi izləyənlər ayrıca yüklənir, sonra müqayisə edilir.
GET /api/v1/friendships/<uid>/followers/
3
Mərhələ 3 — Blok Siyahısı
Blok edilmiş hesablar toplanır. API uğursuz olarsa DOM scraping aktivləşir.
GET /api/v1/users/blocked_list/
✓
Skan Tamamlandı
Nəticələr localStorage-ə yazılır, əvvəlki skan ilə müqayisə edilir, dəyişikliklər göstərilir.

Fetch Interceptor
window.fetch override ilə Instagram-ın öz sorğularını passiv şəkildə dinləyir. Əlavə sorğu göndərilmir — məlumatlar Instagram yüklədikcə avtomatik toplanır.
→ /blocked_list/ — blok siyahısı
→ hide_story — gizlədilmişlər
→ /comments/ — top fans üçün
→ web_profile_info — inaktiv hesablar
Instagram API-ları

İstifadə Edilən API-lar

Bütün sorğular credentials: 'include' ilə Instagram session cookie-sini istifadə edir.

Metod Endpoint Məqsəd Status

Məlumat Saxlama

localStorage Açarları

Bütün məlumatlar yalnız öz brauzerinizdə saxlanır. Heç bir server yoxdur.

sayn_history_v1
Son 10 skan snapshotı. Yeni/itirilmiş izləyicilər müqayisəsi üçün.
sayn_blocked_all_v1
Tam blok siyahısı cache-i. API-dan alınan bütün bloklananlar.
sayn_hidden_story_v1
Story gizlədilmiş istifadəçilər cache-i.
sayn_wl_v3
Whitelist — unfollow edilməyəcək istifadəçilər [{id, username}].
sayn_notes_v2
İstifadəçi qeydləri {userId: "mətn"} formatında.
sayn_likers_v1
Post şərh edənlər cache-i {postId: users[]} — Top Fans üçün.
sayn_cfg_v3
Konfiqurasiya parametrləri — skan gecikmələri, fasilələr.
sayn_pro_v1
PRO statusu (boolean) + sayn_license_v1 lisenziya açarı.
